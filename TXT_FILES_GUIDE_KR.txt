================================================================================
                    ATM 시뮬레이터 TXT 파일 가이드
================================================================================

개요
----
이 가이드는 ATM 시뮬레이터에서 사용되는 텍스트 파일의 목적과 형식을 설명합니다.

================================================================================
파일 유형
================================================================================

1. 초기 조건 파일 (입력)
------------------------
목적: 초기 시스템 데이터 로드 (은행, 계좌, ATM)
예시: sample_initial_condition.txt

형식 구조:
- 1번째 줄: 전체 개수 (B = 은행, A = 계좌, N = ATM)
- 2번째 줄 ~ (B+1)번째 줄: 은행 이름 (한 줄에 하나씩)
- (B+2)번째 줄 ~ (B+1+6A)번째 줄: 계좌 정보 (계좌당 6줄)
- (B+2+6A)번째 줄 ~ 끝: ATM 정보 (ATM당 5줄)

상세 형식:
1번째 줄:              <B> <A> <N>

2 ~ (B+1)번째 줄:      <은행이름>
                       <은행이름>
                       ...

(B+2) ~ (B+1+6A)번째 줄:   [각 계좌마다 - 총 6줄]
                       <은행이름>
                       <사용자이름>
                       <계좌번호>
                       <사용가능금액>
                       <카드번호>
                       <카드비밀번호>

(B+2+6A) ~ 끝:             [각 ATM마다 - 총 5줄]
                       <주거래은행이름>
                       <일련번호>
                       <유형>
                       <언어>
                       <초기자금>

주의: 각 필드는 반드시 별도의 줄에 있어야 합니다. 1번째 줄과 초기자금 줄을 
제외하고는 한 줄에 여러 값을 넣지 마세요!

필드 설명:
- B, A, N: 정수 개수 (1번째 줄에만 공백으로 구분)
- 은행이름: 은행 이름 (예: Kakao, Daegu)
- 사용자이름: 사용자 이름 (중복 허용, 형식 자유)
- 계좌번호: 계좌 번호 (형식 자유, 권장: XX-XX-XXXXXX)
- 사용가능금액: 계좌 시작 잔액 (정수)
- 카드번호: 카드 번호 (형식 자유, 권장: XXXX-XXXX-XXXX)
- 카드비밀번호: 계좌 비밀번호 (형식 자유)
- 주거래은행이름: 이 ATM이 속한 은행
- 일련번호: ATM 일련번호 (형식 자유, 권장: 6자리 숫자)
- 유형: 
  * Single = 주거래 은행 계좌만 접근 가능
  * Multi = 모든 은행 계좌 접근 가능 (타행 거래 허용)
- 언어: 다음 중 하나 선택
  * Unilingual = 영어만
  * Bilingual = 영어와 한국어 지원
- 초기자금: 공백으로 구분된 4개의 정수 (지폐 개수)
  * 순서: <5만원권> <1만원권> <5천원권> <천원권>

예시 (sample_initial_condition.txt):
2 2 2
Kakao
Daegu
Kakao
Chris
100-100-111111
4000
1111-1111-1111
1234
Daegu
Andor
200-200-222222
6000
2222-2222-2222
4321
Kakao
000111
Single
Unilingual
2 3 1 3
Daegu
000222
Multi
Bilingual
0 0 0 60

줄 수 계산:
총 줄 수 = 1 + B + (6 × A) + (5 × N)
위 예시의 경우: 1 + 2 + (6 × 2) + (5 × 2) = 25줄

2. 거래 내역 파일 (출력)
-------------------------
목적: 관리자 메뉴에서 거래 기록 내보내기

형식:
/
This ATM sessions: <총횟수> (customer <고객횟수>, admin <관리자횟수>)
========================================
        TRANSACTION HISTORY            
========================================
ID: <거래ID>
  ATM: <ATM일련번호>
  Card: <카드번호>
  Type: <거래유형>
  Amount: <금액>
  Fee: <수수료>
  From: <은행이름> / <계좌번호>
  [To: <은행이름> / <계좌번호>]  // 이체의 경우만
  Note: <설명>
----------------------------------------
...
========================================

거래 유형:
- Deposit: 계좌 입금
- Withdrawal: 계좌 출금
- AccountTransfer: 계좌 간 이체
- CashTransfer: 현금 수수료 지불 이체

예시:
/
This ATM sessions: 2 (customer 1, admin 1)
========================================
        TRANSACTION HISTORY            
========================================
ID: 1
  ATM: 000001
  Card: 1111-1111-1111
  Type: Deposit
  Amount: 50000
  Fee: 0
  From: Shinhan / 100-100-111111
  Note: Deposit completed
----------------------------------------
ID: 2
  ATM: 000001
  Card: 1111-1111-1111
  Type: AccountTransfer
  Amount: 100000
  Fee: 2000
  From: Shinhan / 100-100-111111
  To: KB / 200-200-222222
  Note: Account transfer completed (fee deducted from source account)
----------------------------------------
========================================

================================================================================
사용 방법
================================================================================

초기 데이터 로드:
1. 프로그램 실행: ./atm
2. 프롬프트에서 초기 조건 파일 경로 입력
3. 파일이 유효하지 않거나 찾을 수 없으면 기본 샘플 데이터가 로드됩니다

거래 내역 내보내기:
1. 관리자 카드 자격 증명으로 관리자 세션 시작
2. 옵션 [2] 거래 내역을 파일로 내보내기 선택
3. 원하는 출력 파일 이름 입력 (예: output.txt)
4. 파일이 현재 디렉토리에 저장됩니다

================================================================================
참고 사항
================================================================================
- 모든 거래 내역 파일은 영어로만 내보내집니다 (언어 중립)
- 수수료 정보는 모든 거래 유형에 포함됩니다
- 초기 조건 파일은 시작 시 전체 시스템 상태를 결정합니다
- sample_initial_condition.txt를 사용자 정의 구성 생성 참조로 보관하세요
- 파일 입출력 예시는 file_input.cpp를 참조하세요 (제공된 경우)
